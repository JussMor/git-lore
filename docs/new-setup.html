Antes de Implementar

Workflow de Git-Lore para equipos con AI

Vista previa del flujo endurecido: la IA autocompleta mensajes de tools, pero nunca escribe sin estado fresco ni sin pasar preflight.

State-First obligatorio Preflight-First obligatorio Sync idempotente Transiciones auditables
Fase A

Contrato y reglas

  • Definir gates de edicion y commit
  • Modelo de estados y transiciones validas
  • Politica de autofill por IA
Fase B

Capas MCP core

  • state_snapshot con TTL
  • memory_preflight con block/warn/info
  • memory_search con ranking local
Fase C

Control operacional

  • set-state con motivo auditado
  • sync idempotente y dedupe
  • alinear merge driver con capacidades reales
Fase D

Flujo estandar

  • state_snapshot antes de escritura
  • preflight antes de editar/commit/sync
  • autofill IA con confirmacion opcional
Fase E

Calidad y cierre

  • tests unitarios e integracion multi-agente
  • pruebas end-to-end en .test
  • actualizacion de docs y matriz de cumplimiento

Gate 1: Antes de editar

Obligatorio para signal, propose y cualquier accion de escritura.

state_snapshot vigente soft-lock PRISM hard-lock detectado

Gate 2: Antes de commit y sync

Evita deriva de estados, secretos y crecimiento artificial de memoria.

validacion scripts ok entropia alta sync no idempotente

Pipeline objetivo (semi-autonomo)

1. state_snapshot
2. memory_preflight
3. signal
4. context + memory_search
5. propose
6. edit
7. validate
8. checkpoint
9. commit
10. sync
Operacion de escritura permitida solo si Gate 1 y Gate 2 estan en verde
